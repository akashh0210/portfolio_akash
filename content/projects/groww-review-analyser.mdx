## Problem

App review analysis is a PM's least favourite chore: high volume, low signal density, and by the time you've read 300 reviews you've forgotten the first 200. App Store and Play Store pages accumulate hundreds of reviews weekly — extracting product signals from them manually is a full afternoon's work.

## Hypothesis

A structured pipeline — scrape, cluster, summarise, prioritise — could reduce time-to-insight from hours to under a minute while surfacing themes a manual read would miss (e.g. a specific notification bug appearing in 12% of 1-star reviews).

## What I built

A Node.js 20 TypeScript + FastAPI pipeline scrapes App Store and Google Play reviews, passes them through a Groq-powered clustering step (theme extraction and severity classification using `llama-3.3-70b`), and generates a Streamlit dashboard with PM-ready insight cards. A GitHub Actions workflow automates weekly runs and delivers reports via Markdown, email drafts, and Google Docs through an MCP integration.

An MCP integration lets Claude query the insight database conversationally — "what are the top 3 complaints this week?" returns a structured answer in seconds. The tool includes automatic PII scrubbing at ingestion to keep user data clean. Deployed on Hugging Face Spaces.

## Outcome & learning

5,000+ reviews processed in under 60 seconds. The MCP integration was the highest-leverage addition — it turned a reporting dashboard into a conversational tool, which made it 10x more likely to actually be used. Lesson: build the interface your user will actually open, not the one that's easiest to ship.
